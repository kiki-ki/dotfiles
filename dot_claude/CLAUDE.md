# Global Rules

- Always create PRs as draft (`gh pr create --draft`)
- When creating issues or PRs with `gh`, do not mention users or teams (no `@username`, `@org/team`). Escape with backticks or rephrase if a handle must appear in the body
- Never commit secrets or credentials
- Follow Conventional Commits: `type(scope): summary`
- Do not add comments that restate the code. Only add comments to explain "why", not "what"
